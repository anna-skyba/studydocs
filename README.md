# StudyDocs AI

StudyDocs AI is a simple web application for chatting with uploaded PDF documents.

The user can upload PDF files, ask questions about their content, and receive answers generated by a local LLM. The application uses RAG: it extracts text from PDFs, splits it into chunks, creates embeddings, stores them in PostgreSQL with pgvector, and sends the most relevant chunks to the LLM.

## Technologies

- Java
- Spring Boot
- Spring JDBC
- Spring AI
- PostgreSQL
- pgvector
- Ollama
- HTML, CSS, JavaScript
- Docker

## How to Run

### 1. Start PostgreSQL

Run Docker:

```bash
docker compose up -d
```

Database runs on:

```text
localhost:55432
```

### 2. Start Ollama models

Install Ollama and pull the required models:

```bash
ollama pull llama3.2:3b
ollama pull nomic-embed-text
```

Check models:

```bash
ollama list
```

### 3. Add database configuration

Create file:

```text
secrets/postgresql.properties
```

Example:

```properties
spring.datasource.url=jdbc:postgresql://localhost:55432/studydocs_db
spring.datasource.username=studydocs_user
spring.datasource.password=studydocs_password
spring.datasource.driver-class-name=org.postgresql.Driver
```

The `secrets` folder should not be committed to Git.

### 4. Run Spring Boot

Start the project from IntelliJ IDEA or with Maven.

Open the application:

```text
http://localhost:8080
```

## Main Features

- Upload PDF documents
- Extract text from PDFs
- Store document chunks in PostgreSQL
- Store embeddings in pgvector
- Ask questions about uploaded documents
- Show answer sources
- Open uploaded PDF files
- Delete documents
- Simple web interface

## How to Improve LLM Answers

The quality of answers can be changed in several places.

### 1. Change number of retrieved chunks

File:

```text
SemanticSearchService.java
```

Change:

```java
.topK(4)
```

Higher value gives the LLM more context:

```java
.topK(6)
```

Lower value gives shorter and more focused context:

```java
.topK(2)
```

### 2. Change the prompt

File:

```text
ChatService.java
```

The system prompt controls how the LLM answers.
For example, it can be changed to make answers shorter, more detailed, or stricter.

Example rule:

```text
Use only the provided context. Do not invent facts.
```

### 3. Change chunk size

File:

```text
TextChunkingService.java
```

Current chunk settings can be adjusted:

```java
CHUNK_SIZE = 1000
OVERLAP = 150
```

Larger chunks give more context.
Smaller chunks can make search more precise.

### 4. Use a better local model

File:

```text
application.properties
```

Chat model:

```properties
spring.ai.ollama.chat.model=llama3.2:3b
```

A stronger model may produce better answers, but it needs more computer resources.

### 5. Reupload documents after changes

After changing chunking or embeddings, old vectors should be deleted and PDFs uploaded again.

Example cleanup:

```sql
DELETE FROM vector_store;
DELETE FROM document_chunks;
DELETE FROM study_documents;
```

Then upload the PDF files again.

## Notes

The project works best with normal PDFs that contain selectable text.

Scanned PDFs or image-only PDFs may need OCR support in the future.
